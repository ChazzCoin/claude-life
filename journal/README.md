# Journal

Daily entries. Format: `journal/YYYY/MM/DD.md`. One file per day.

---

## Why this is separate from `docs/notes/`

- **`journal/`** is *processing in time*. Today's thinking
  about today. Raw, fast, chronological. Sometimes private
  even from itself.
- **`docs/notes/`** is *durable lessons*. Things I've learned
  that future-me needs. Curated. Indexed.
- **`memories/`** is *durable facts*. People I know, events
  that happened, places that matter. Reference, not
  reflection.

A journal entry might *promote* into a note (when a lesson
emerges) or a memory (when a fact wants to stick). The
promotion is conscious — happens during `/lessons`,
`/weekly-review`, or in-session via `/codify`.

## Structure

- `journal/2026/05/07.md` — today
- `journal/2026/05/06.md` — yesterday
- ...

Each daily file is freeform markdown. No required template.
Common shapes:
- Brain dump
- Reflection on something specific
- A conversation with Claude that's processing-shaped, not
  task-shaped
- Gratitude (or it lives in `docs/gratitude/` if you prefer it
  separate)

## Privacy

Journal is the most-private layer. Default assumption: nobody
ever reads this except future-me. The `/publish` projection
should *never* surface journal contents.
